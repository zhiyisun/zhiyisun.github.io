# Implementation Plan: Migrate Blog to Minimal Mistakes Theme

**Branch**: `001-migrate-minimal-mistakes` | **Date**: 2026-03-19 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/001-migrate-minimal-mistakes/spec.md`

## Summary

Migrate the personal blog from the lanyon/poole theme to the minimal-mistakes Jekyll theme while preserving all 17 existing posts and the about page content. The migration involves replacing theme files, updating configuration, and ensuring content renders correctly with the new theme's layout and styling.

**Key Decisions from Clarification Session**:
- Theme skin: Use alternate built-in skin (not default), selection deferred to visual preview
- Analytics: Google Analytics tracking via `_config.yml`
- 404 Page: Use minimal-mistakes default layout
- Git History: Preserve all commits, remove lanyon remote only

## Technical Context

**Language/Version**: Ruby 2.7+ (Jekyll runtime)
**Primary Dependencies**: Jekyll 4.x, minimal-mistakes-jekyll gem, jekyll-feed (for RSS)
**Storage**: Static files (Markdown posts, YAML configuration)
**Testing**: Manual verification via `jekyll serve` local preview
**Target Platform**: GitHub Pages (static site hosting)
**Project Type**: Static blog/site
**Performance Goals**: Homepage loads within 3 seconds
**Constraints**: Must preserve all existing content; maintain URL structure; preserve git history
**Scale/Scope**: 17 posts, 1 static page (About), single-author blog
